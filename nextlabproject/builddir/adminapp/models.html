Admin App models¶

class adminapp.models.AppDetails(*args: Any, **kwargs: Any)¶

Bases: Model

AppDetails model is used to store app details that are being added by admin user. It contains 7 fields id(PK), icons, appName, appLink, category, subCategory, points.

class adminapp.models.LoginDetails(*args: Any, **kwargs: Any)¶

Bases: Model

LoginDetails model is used to store user login details including admin and enduser. It contains two fields username and password. Here ‘username’ is used as Primary Key. It is in a one to many relationship in a TaskDetails model.

class adminapp.models.TaskDetails(*args: Any, **kwargs: Any)¶

Bases: Model

TaskDetails model is used to store user completed completed task details. It contains four fields id(PK), username(ForeignKey), tasksId, points, screenShot. It is in a many to one relationship with LoginDetails model.
